# Portfolio Chatbot Backend (Spring Boot)

AI chatbot API for Tirth Sutariya's portfolio. The knowledge base is Tirth's
resume (`src/main/resources/knowledge/resume-kb.json`).

## Two modes

| `chatbot.use-llm` | Behavior | Needs API key? |
|---|---|---|
| `false` (default) | Keyword retrieval over the resume KB | ❌ No |
| `true` | **Groq LLM RAG** grounded on the resume | ✅ Yes (`GROQ_API_KEY`) |

Both modes stay loyal — the bot only talks about Tirth and deflects off-topic questions.

## Run it

```bash
cd backend
./mvnw spring-boot:run          # or:  mvn spring-boot:run
```

Backend starts on **http://localhost:8090**.

Test:
```bash
curl -X POST http://localhost:8090/api/chat \
  -H "Content-Type: application/json" \
  -d '{"question":"what are his projects?"}'
```

## Enable the LLM (RAG) mode

1. Get a free key at https://console.groq.com
2. Set it and turn LLM on:
   ```bash
   export GROQ_API_KEY=gsk_your_key_here      # Windows: setx GROQ_API_KEY gsk_...
   ```
   Then in `application.properties` set `chatbot.use-llm=true`.
3. Restart. Now answers are generated by Groq (Llama 3.3), grounded on your resume.

## Frontend

`js/chatbot.js` already points here:
```js
const CONFIG = { useBackend: true, endpoint: 'http://localhost:8090/api/chat' };
```
If the backend is offline, the frontend falls back to its built-in KB automatically.

## Endpoints

- `POST /api/chat` → `{ "question": "..." }` ⇒ `{ "answer": "...", "fromLlm": false }`
- `GET  /api/health` → `{ "status": "up" }`

## Deploy

Deploy on Render / Railway / Fly.io, then:
1. Update `chatbot.cors.allowed-origins` with your portfolio's domain.
2. Update `CONFIG.endpoint` in `js/chatbot.js` to the deployed API URL.
